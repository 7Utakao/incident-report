# 基本方針書（Incident Report System）

## 目的
SES会社の社内利用を目的としたインシデント管理システムを開発する。
チームや個人が躓いた事や原因を記録・可視化し、ナレッジとして共有することで以下を実現する：
- レポート提出を通じた知見共有
- 不適切・機密情報を含まない安全な情報蓄積
- 可視化による自己学習・他者理解・チーム改善

## 対象ユーザー
- SES会社に所属するエンジニア、リーダー、マネージャー
- 特に現場での躓きを共有し、他者の知見を活用したい人

## 入力に含めたくない情報
- 実在の個人名
- 顧客名、顧客企業名
- 住所・電話番号・メールアドレスなどの個人情報
- 案件固有の機密（契約条件、コード断片、設計図の抜粋など）

## 原因カテゴリ（例）
- 技術的知識不足
- 環境設定ミス
- コミュニケーション不足
- 工数/スケジュール管理
- 要件定義・仕様理解
- その他

## 出力形式（AIの応答）
- すべて JSON 形式で返す
- 例：
{
  "summary": "短い要約文",
  "tags": ["環境設定", "データベース"],
  "category": "環境設定ミス",
  "anonymizedText": "匿名化後の本文",
  "suggestedReplacements": [
    {"from": "顧客名A", "to": "[顧客名]"}
  ]
}

## 画面一覧
1. ホーム
   - マスコット表示（提出数で成長）
   - 本日/今週の提出数
2. レポート一覧
   - タグ/カテゴリ/期間/キーワードで検索・絞り込み
3. レポート詳細
   - 本文・要約・タグ・カテゴリ表示
   - 投稿者名は匿名化
4. レポート新規作成
   - 入力フォーム
   - 提出前にAIによる不適切/機密チェック
   - 要約・タグ・カテゴリをAI提案
5. ダッシュボード
   - カテゴリ分布グラフ
   - 時系列グラフ
   - 上位タグランキング
   - 所感（AIによる一段落のまとめ）
6. マイページ（任意）
   - 自分の提出数、ポイント、獲得バッジなど
7. 管理画面（任意）
   - 禁止ワードリスト編集
   - 監査ログ確認
